课件里的表达式层次

<表达式> ::= <项> { (+|-) <项> } <项> ::= <因子> { (*|/) <因子> } <因子> ::= id | num | '(' <表达式> ')'
为了让“压栈 / 出栈”一格一格可见，演示时把上面的 EBNF 暂时展开成普通 BNF：
E → T E' E' → + T E' | - T E' | ε T → F T' T' → * F T' | / F T' | ε F → id | num | ( E )

这一步到底在证明什么

1. 当前 sym 在驱动分支。 看的是当前输入，而不是整串未来。
2. getsym 在推动流程。 只有匹配了终结符，输入指针才前进。
3. 栈在保存“还没完成的结构”。 例如看到 ( 以后，分析器会把“以后还要补一个 ) ”这件事记在栈里。
4. 所以语法分析对应下推能力。 不是只靠有限状态乱猜，而是靠“当前状态 + 一个栈”来工作。

如何在你的这套 PPT 里接上这页演示

先接在“表达式的 EBNF”与“expr / term / factor 伪代码”之后。学生已经熟悉表达式分层，此时最容易把 递归调用栈 和 PDA 栈 对起来。
第一遍只放 (1+2)*3。因为它同时展示了：括号、优先级、嵌套返回、以及为什么需要栈。
第二遍再换 a+b*c。让学生看到：就算没有括号，表达式分层依然会把“加减”和“乘除”分到不同层里。
如果你要提问，可以固定问四句：现在 sym 是谁？getsym 有没有发生？调用栈顶是谁？PDA 栈顶是谁？ 这样学生会逐渐形成结构感。
