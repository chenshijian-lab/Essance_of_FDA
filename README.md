# Essance_of_FDA

这是给《编译原理》同学课后使用的自动机可视化小仓库。  
你可以把它当成“动态版课本”：不是只看定义，而是直接看到状态怎么走、栈怎么变。

本仓库有两个 HTML 演示：

- `automata_spirit_visualization_v2.html`
- `pl0_pda_visualization.html`

---

## 你能学到什么？

这两个文件对应编译原理里两个关键问题：

1. **词法阶段：自动机如何识别输入？**
2. **语法阶段：为什么要用栈（PDA）来处理结构？**

如果你对“状态转移图”“下推自动机”“语法分析栈”总觉得抽象，这两个页面就是拿来帮你建立直觉的。

---

## 先跑哪个？顺序建议

请按下面顺序：

1. `automata_spirit_visualization_v2.html`  
2. `pl0_pda_visualization.html`

原因：先理解“状态机按字符工作”，再理解“栈如何支撑语法结构”。

---

## 1) automata_spirit_visualization_v2.html

### 它在讲什么

- 输入串逐字符读取
- 当前状态如何变化
- 何时进入接受态 / 非接受态

### 你该怎么用

1. 先自己猜：下一步会跳到哪个状态
2. 再点按钮运行，看是否猜对
3. 换一个输入，重复上面步骤
4. 特别试试“错误输入”，看它在哪一步失败

### 你要带走的结论

- 词法分析器不是“魔法”，本质是自动机在跑
- 每读一个字符，状态都会更新
- “识别 token”可以理解成“到达某类可接受状态”

---

## 2) pl0_pda_visualization.html

### 它在讲什么

- 栈顶符号如何决定下一步动作
- 匹配终结符、展开非终结符的过程
- 语法分析为什么需要“记住未完成结构”

### 你该怎么用

1. 看每一步前，先盯住“栈顶”
2. 预测接下来是“匹配”还是“展开”
3. 跑一步，核对你的预测
4. 把输入改错一个符号，观察报错点

### 你要带走的结论

- 语法分析的核心不是背规则，而是“栈驱动过程”
- 程序的层次结构（尤其块结构）需要栈来维护上下文
- 这和后面学的递归下降/预测分析是同一件事的不同表达

## 课后最推荐的练习（10~20 分钟）

### 练习 A（自动机）

- 选 3 组输入
- 每组先写下你预测的状态路径
- 再运行验证，记录哪里预测错了

### 练习 B（PDA）

- 选 1 组正确输入 + 1 组错误输入
- 对比两组在“栈变化”上的差异
- 写 3 句话总结：错误是如何暴露出来的

## 常见问题

### Q1: 打开后空白或动画不动？

先换一个浏览器（Chrome/Edge），再刷新页面。

### Q2: 看不懂每一步在干嘛？

别急着“自动播放”，先一步一步点，重点看：

- 当前状态 / 栈顶
- 输入头位置
- 本步动作说明

### Q3: 我应该记住什么？

记住这句就够了：

> 编译不是黑箱，它是“状态 + 栈 + 规则”一步一步执行出来的过程。
