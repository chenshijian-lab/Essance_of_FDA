# Essance_of_FDA

面向《编译原理》课堂的自动机可视化教学仓库。  
本仓库当前包含两个 HTML 交互演示，分别对应编译系统教学中的两个关键层面：

- 词法分析背后的“有限状态自动机思想”
- 语法分析背后的“栈 + 状态迁移（下推自动机/PDA）思想”

结合 `2-PL0编译系统-2.pptx` 的课程脉络（从 PL/0 编译流程到词法/语法机制），这两个演示可以用于“先建立动态直觉，再讲形式定义”。

---

## 1) `automata_spirit_visualization_v2.html`

### 教学意义（对应词法分析前置直觉）

这个演示的核心价值，不在于让学生背定义，而在于让学生“看到自动机怎么工作”：

- 输入串是如何被逐字符扫描的
- 状态是如何沿着边迁移的
- 什么时候进入接受态/拒绝态
- 为什么“正规模式识别”可以被转化为有限状态过程

在讲 PL/0 编译系统时，学生常把“词法分析器”当黑盒。这个页面可以把黑盒打开，让学生理解：

> 词法分析器本质上就是一个按字符流驱动的自动机执行器。

### 课堂建议用法

- 在讲“单词符号切分”之前先演示 3~5 个输入样例
- 让学生预测下一步状态迁移，再点击执行
- 用“错误输入”展示自动机失败路径（比口头解释更有效）

---

## 2) `pl0_pda_visualization.html`

### 教学意义（对应语法分析与 PL/0 块结构）

这个演示更贴近 PL/0 语法层，突出“栈式控制”的语法处理思想：

- 栈顶符号如何驱动下一步动作
- 终结符匹配与非终结符展开如何交替进行
- 为什么语法分析需要记忆“尚未完成的结构”
- 在块结构（`begin...end`、过程嵌套）下，栈机制如何保持上下文

它对应 PL/0 编译教学中的核心过渡：

> 从“字符串是否被接受”走向“程序结构如何被系统化识别与展开”。

学生看完后，通常更容易理解递归下降/预测分析中的“调用栈感”。

### 课堂建议用法

- 配合 PL/0 语法片段逐步播放（不要一次跑完）
- 重点让学生观察“栈变化”而不是只看结果
- 把某个输入故意改错，观察在哪一步失配

---

## 建议教学顺序（与 PL/0 编译系统章节衔接）

1. 先用 `automata_spirit_visualization_v2.html` 建立词法自动机直觉  
2. 再用 `pl0_pda_visualization.html` 过渡到语法栈机制  
3. 最后回到 PL/0 编译流程图，连接“词法 → 语法 → 中间表示/目标代码”

这样学生会形成从“状态机识别”到“结构化语法处理”的连续认知链，而不是碎片化记忆。

---

## 运行方式

两个文件都是纯 HTML 交互页面，直接浏览器打开即可：

- `automata_spirit_visualization_v2.html`
- `pl0_pda_visualization.html`
